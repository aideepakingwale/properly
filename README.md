Properly is a full-stack web application that listens to a child read aloud, analyses their pronunciation at the **phoneme level** using Microsoft Azure Pronunciation Assessment, and delivers warm, intelligent coaching feedback — making expert phonics support accessible to every family at a fraction of the cost of private tutoring.

Built entirely out of passion by **Deepak Ingwale** (Solution Architect, 20 years in IT) on the occasion of his 18-year anniversary at Tata Consultancy Services — to give every child the gift of reading. 📖

---

## ✨ Features

### 🎙️ Real-Time Pronunciation Assessment
- **Azure Speech SDK** streams audio over WebSocket for phoneme-level scoring
- Dual-path assessment: browser SDK → REST API (3 retries) → Groq Whisper → text comparison
- Per-word and per-phoneme accuracy scores with error type classification (Omission / Mispronunciation)
- Wrong-sentence detection using Jaccard similarity — prevents accidental 100% scores
- Auto-stop at 85% words captured or 15-second hard cap with elapsed timer

### 📖 Personalised AI Stories
- **Gemini 2.5 Flash** + **Groq Llama 3.1 70B** generate bespoke stories per child
- Tailored to child's name, phonics phase, and chosen interests
- 16 story themes: space, dragons, animals, adventure, and more
- Phase-appropriate vocabulary aligned to the **DfE Letters and Sounds** framework (Phases 2–6)

### 📚 Illustrated Phonics Books
- AI-generated illustrations (HuggingFace FLUX.1-schnell → Pollinations → Picsum fallback)
- **PDFKit** generates A4 books with:
  - Colour-coded grapheme tiles per word
  - IPA phoneme strip footer (`/ð//ɛ/ /f//æ//t/ /k//æ//t/ /s//æ//t/`)
  - QR code on back cover linking to the book in-app
- Download as PDF · Delete with R2 cleanup · Report issue (attaches generation log)

### 🔤 Interactive Phonics Guide
- Full DfE Phase 2–6 curriculum explorer with real Azure phoneme sounds
- **Alphabet Sound Explorer** — 26 letters, each plays its real phoneme (not letter name)
- Step-by-step blending demos: tap any grapheme tile to hear its isolated sound
- Phoneme-by-phoneme breakdown of phonics terms themselves (e.g. "Phoneme" → /f/ /əʊ/ /n/ /iːm/)
- 🎤 Mic practice with Azure PA scoring on every word

### 🦉 Mrs Owl AI Coach
- Post-reading coaching tips generated by Gemini/Groq AI
- Phase-aware feedback targeting the child's specific phoneme errors
- Azure Neural TTS (`en-GB-SoniaNeural`) delivers tips in a warm British voice

### 🏆 Gamification & Engagement
- Acorn rewards 🌰 earned per session based on accuracy score
- Daily reading streaks 🔥 displayed on child home screen
- 15 achievement trophies unlocked by reading milestones
- Acorn shop for avatar accessories and digital stickers
- Confetti celebration on 100% accuracy

### 👨‍👩‍👧 Parent Dashboard
- Per-child progress tracking: accuracy trends, words read, phase, streaks
- Phase management — promote child when they master a level
- Book library with download, delete, and reorder

### 🛡️ Admin Panel
- User management, book credit grants
- Content reports queue with generation debug log attachment
- Responsive light-theme UI (desktop + mobile)

### Speech Assessment Pipeline

```
MediaRecorder (WebM audio)
  │
  ├─ Web Speech API ──── quality gate (2.5s) ───────────────┐
  │                                                         │
  └─ Azure Speech SDK WebSocket ─── primary PA              │
        │                                                   │
        ├─ Success: per-phoneme scores ✅                   │
        └─ Failure (×3 retry):                             │
              └─ REST API → Azure PA → Groq Whisper ◄───────┘
                    └─ Jaccard similarity gate
                          └─ Text comparison fallback
```

## 🚀 Getting Started

### Prerequisites

- **Node.js 22+**  (`node --version`)
- **ffmpeg** — `brew install ffmpeg` (macOS) or `apt install ffmpeg` (Ubuntu)
- Accounts for the external services listed below

### 1. Clone and install

```bash
git clone https://github.com/deepakingwale/properly-phonics.git
cd properly-phonics

# Install all three services at once
npm run install:all

# Or individually
cd backend  && npm install
cd frontend && npm install
cd admin    && npm install
```

### 2. Environment variables

```bash
cp backend/.env.example backend/.env
```

Edit `backend/.env`:

```env
# ── REQUIRED ──────────────────────────────────────────────────────────────────

# Auth — generate with: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
JWT_SECRET=your-64-char-secret-here
JWT_EXPIRES_IN=30d

# Azure Cognitive Services — portal.azure.com → Create resource → Speech
AZURE_SPEECH_KEY=your-azure-speech-key
AZURE_SPEECH_REGION=uksouth

# AI — at least one required
GEMINI_API_KEY=your-gemini-key        # aistudio.google.com → Get API Key (free)
GROQ_API_KEY=your-groq-key            # console.groq.com → API Keys (free)

# Cloudflare R2 — dash.cloudflare.com → R2 → Create bucket "properly"
R2_ACCOUNT_ID=your-cloudflare-account-id
R2_ACCESS_KEY_ID=your-r2-access-key-id
R2_SECRET_KEY=your-r2-secret-key
R2_BUCKET=properly

# Admin access (comma-separated emails auto-promoted to admin on login)
ADMIN_EMAILS=your@email.com

# CORS
CORS_ORIGINS=http://localhost:5173

# ── OPTIONAL (image quality for books) ────────────────────────────────────────

# HuggingFace — huggingface.co/settings/tokens → New token (free)
HUGGINGFACE_TOKEN=hf_...

# Pollinations — pollinations.ai → Settings → API Key (free)
POLLINATIONS_TOKEN=...

# ── OPTIONAL (email + payments) ───────────────────────────────────────────────
RESEND_API_KEY=re_...
STRIPE_SECRET_KEY=sk_...
```

### 3. Run locally

```bash
# Start all three in parallel
npm run dev

# Or in separate terminals:
cd backend  && npm run dev    # API  → http://localhost:3001
cd frontend && npm run dev    # App  → http://localhost:5173
cd admin    && npm run dev    # Admin→ http://localhost:5174
```

Open `http://localhost:5173` → **Register** → **Add a child** → **Start reading!**

## ☁️ Deploy to Render.com

The repo includes a complete `render.yaml` Infrastructure-as-Code blueprint.

### Option A: One-click (Blueprint)

1. Fork this repo
2. Go to [dashboard.render.com](https://dashboard.render.com) → **New** → **Blueprint**
3. Connect your forked repo — Render reads `render.yaml` and creates all three services
4. Set the required environment variables in the Render dashboard
5. Click **Deploy** — first build takes ~3–5 minutes

### Option B: Manual

Create three services manually on Render:

| Service | Type | Root Dir | Build | Start |
|---|---|---|---|---|
| `properly-api` | Web Service (Node) | `backend` | `npm install` | `npm start` |
| `properly-web` | Static Site | `frontend` | `npm install && npm run build` | — |
| `properly-admin` | Static Site | `admin` | `npm install && npm run build` | — |

Set `VITE_API_URL` on both static sites to your API's URL: `https://properly-api.onrender.com/api`

> **Free tier note:** Render free services spin down after 15 minutes idle. First request after sleep takes ~30s. Upgrade to Starter ($7/mo) for always-on.

## 🔐 Security

- JWT authentication with 30-day TTL
- bcrypt password hashing (12 salt rounds)
- Helmet.js with 14 security headers
- CORS whitelist via `CORS_ORIGINS` environment variable
- Rate limiting: 200 requests per 15 minutes per IP
- Multer: audio files only, 10MB max upload size
- Cloudflare R2 signed URLs expire after 1 hour
- All secrets injected via environment variables — never hardcoded

## 📊 DfE Phonics Curriculum Coverage

| Phase | Focus | Graphemes Covered |
|---|---|---|
| **2** | Simple CVC · consonants · short vowels | `s a t p i n m d g o c k e u r h b f l` |
| **3** | Digraphs · vowel digraphs · trigraphs | `sh ch th ng ai ee oo ar or ur ow oi igh ear air` |
| **4** | Consonant blends · CCVC · CVCC | `bl br cl cr fl fr gr pl st str spr scr` |
| **5** | Alternative spellings · split digraphs | `a_e i_e o_e ay ea ie ue ew oe au aw` |
| **6** | Morphemes · prefixes · suffixes | `-tion -ture -ous -ful -less -ness un- re-` |

The `phonicsEngine.js` module is the **single source of truth** — all 44 IPA phonemes with display colours, example words, phase metadata, and blend component arrays. Used by `usePhonemePlayer`, `PhonicsLearn`, `PhonicsWord`, and `phonicsAnalyser`.
